# Sports Video Ingestion and Preprocessing Pipeline

This repository contains a configurable video ingestion and preprocessing pipeline for sports computer vision workflows. It accepts local video files, folders, and HLS/stream-style inputs, then produces structured frames, metadata, quality reports, scene segmentation outputs, preview videos, and validation reports.

The processed frame outputs can be passed directly into a downstream computer vision model such as YOLO for object detection.

## Main Entry Point

Run the pipeline from:

```text
src/video_processing_analysis_pipeline.py
```

This file calls:

```text
video_pipeline.cli.main()
```

The main processing orchestration is implemented in:

```text
src/video_pipeline/pipeline.py
```

## Installation

Install the Python dependencies:

```bash
pip install -r requirements.txt
```

FFmpeg and FFprobe must also be installed and available on the system `PATH`, because the pipeline uses them for metadata extraction and video validation.

## Running the Pipeline

Default run using the YAML configuration:

```bash
python src/video_processing_analysis_pipeline.py --config configs/video_processing_analysis.yaml
```

Run with a custom input and output folder:

```bash
python src/video_processing_analysis_pipeline.py --config configs/video_processing_analysis.yaml --input dataset/sample.mp4 --output outputs
```

Run on a local HLS playlist:

```bash
python src/video_processing_analysis_pipeline.py --config configs/video_processing_analysis.yaml --input "C:\Users\ASUS\Downloads\tu_d\real_team_sports_scene_change\hls\playlist.m3u8" --output outputs
```

If the input is a folder and videos are inside subfolders, add:

```bash
--recursive
```

## Configuration

The main configuration file is:

```text
configs/video_processing_analysis.yaml
```

It controls:

- input source and supported video formats
- output directory
- frame sampling mode and target FPS
- target resize resolution
- RGB and grayscale output
- frame quality thresholds
- scene-change detection settings
- processed video and preview generation
- output validation options
- logging level

## Computer Vision Integration with YOLO

The assessment requires that the processed output is usable by a downstream computer vision service or model. This project demonstrates that by running YOLO object detection on the RGB frames generated by the pipeline.

Sample basketball data can be taken from the [PlayOn basketball analysis samples repository](https://github.com/playon/basketball-analysis-samples).

The object detection model used is:

```text
yolov8n.pt
```

This is the lightweight nano YOLOv8 model from [Ultralytics YOLO](https://github.com/ultralytics/ultralytics).

Install YOLO separately:

```bash
pip install ultralytics
```

Example YOLO command used for detection:

```bash
yolo predict model=yolov8n.pt source="/media/cv/HDD/Throwing_Dataset/TU_D/video_ass/output/mix/playlist/frames/good/segment_021/rgb" device=0 conf=0.25 save=True save_txt=True save_conf=True project="/media/cv/HDD/Throwing_Dataset/TU_D/video_ass/output/lol" name="good_seg21"
```

For this repository structure, the equivalent local command is:

```bash
yolo predict model=yolov8n.pt source="outputs/playlist/frames/good/segment_021/rgb" device=0 conf=0.25 save=True save_txt=True save_conf=True project="outputs/yolo" name="good_seg21"
```

Use `device=cpu` instead of `device=0` if a CUDA GPU is not available.

The YOLO output folder contains annotated prediction images and label text files. This confirms that the pipeline output is directly consumable by a computer vision inference workflow.

## Example End-to-End Flow

1. Run the ingestion pipeline on a sports video or HLS playlist.
2. Inspect `outputs/<video_id>/manifests/validation_report.json` and confirm `PASS`.
3. Select a good frame segment, for example:

```text
outputs/playlist/frames/good/segment_021/rgb
```

4. Run YOLO object detection on that folder.
5. Review annotated detection outputs and saved label files.
